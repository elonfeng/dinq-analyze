# 用户验证系统（业务说明）

本目录描述用户验证系统的业务目标、流程与接口对接方式，帮助前后端与产品统一验证规则。

## 业务目标
- **保证用户真实性**：通过分步骤采集与验证信息
- **控制访问权限**：未验证用户限制访问关键分析能力
- **提升履约质量**：为人才/招聘场景提供可用画像

## 用户类型
系统支持两类用户：
- `job_seeker`（求职者）
- `recruiter`（招聘方）

## 业务流程摘要
1) 用户提交基础信息  
2) 逐步完成教育/经历/联系方式等验证步骤  
3) 通过后获得完整权限（受 `require_verified_user` 等验证门控约束）

## 对接建议
- 前端：按步骤驱动表单与状态机
- 后端：统一校验与状态落库
- 产品：配置不同用户类型的验证路径与字段
